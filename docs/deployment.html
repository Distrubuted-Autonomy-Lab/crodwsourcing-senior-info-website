Deployment of Carebot to the Google Cloud Platform

Guide for deploying the application to GCP using Docker Compose, Docker containers, and the Google Cloud SDK.

Overview

This guide describes deploying the application to the Google Cloud Platform (GCP) and exposes three main services: the Django web application, an Nginx proxy, and a PostgreSQL (PostGIS + pgvector) database.

Prerequisites

  • Google Cloud SDK (gcloud)

Deployment steps

1. Configure Google Cloud SDK CLI

First, we must ensure that we are logged into the account with which the deployment should be associated. If you aren't logged in, run the below script::

gcloud auth login

2. Create and configure the GCP project

Create a project and set it as the default:

gcloud projects create <project-id> --name="<project-name>"
gcloud config set project <project-id>

Attach a billing account (list with gcloud billing accounts list and link with):

gcloud beta billing projects link <project-id> --billing-account=<billing-account-id>

Enable Compute Engine:

gcloud services enable compute.googleapis.com

3. Create and configure the virtual machine

Create a startup script to install Docker and Compose on boot. Example startup script:

#!/bin/bash

set -euo pipefail

# Configure startup script logging
LOG=/var/log/startup.log
exec > >(tee -a "$LOG") 2>&1

echo "=== Startup Script Beginning ==="

apt-get update
apt-get install -y apt-transport-https ca-certificates curl gnupg lsb-release

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo \"$VERSION_CODENAME\") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

echo "=== Startup Script Finished ==="

Create the Compute Engine instance referencing the startup script:

gcloud compute instances create <instance-name> \
    --project=<project-id> \
    --zone=us-central1-a \
    --machine-type=e2-medium \
    --image-family=ubuntu-2204-lts \
    --image-project=ubuntu-os-cloud \
    --boot-disk-size=30GB \
    --tags=http-server,https-server \
    --metadata-from-file=startup-script=./<startup-script>

Configure firewall rules (HTTP/HTTPS):

gcloud compute firewall-rules create allow-http \
    --action=ALLOW \
    --rules=tcp:80 \
    --source-ranges=0.0.0.0/0 \
    --target-tags=http-server \
    --description="Enable HTTP traffic"

# Optional (only if HTTPS is configured)
gcloud compute firewall-rules create allow-https \
    --action=ALLOW \
    --rules=tcp:443 \
    --source-ranges=0.0.0.0/0 \
    --target-tags=https-server \
    --description="Enable HTTPS traffic"

4. Prepare Cloud environment variables

Create a .env-gcloud based on .env-gcloud.example and populate it with the required values. Keep this file private.

5. Upload code and environment variables to the instance

Either copy files directly or clone the repo on the instance. Example using gcloud copy:

gcloud compute ssh <instance-name> --command="mkdir -p ~/carebot"
gcloud compute scp --recurse . <instance-name>:~/carebot/
gcloud compute scp .env-gcloud <instance-name>:~/carebot/root/

Or, SSH and clone the repository:

gcloud compute ssh <instance-name>

git clone <repository-url> ~/carebot/
cd ~/carebot/root
# Ensure .env-gcloud is at ~/carebot/root/.env-gcloud

6. Prepare for deployment on the instance

gcloud compute ssh <instance-name>
cd ~/carebot/root/

7. Deploy with Docker Compose

Run the Compose stack with the GCP overlay:

sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml up -d

8. Initialize database and Django

sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml exec web python manage.py migrate
sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml exec web python manage.py createsuperuser

Troubleshooting

Application errors

  1. Ensure API keys are valid and active.
  2. Validate SECRET_KEY and ALLOWED_HOSTS.
  3. Check application logs (see Monitoring below).

Database errors

  1. Confirm the Postgres service is running: docker ps (look for web or root-web).
  2. Ensure DB username/password match between web and the Postgres service.

Security & reliability

  • Keep .env-gcloud private.
  • Restrict access with GCP firewall rules.
  • Regularly update Docker images and apply OS patches.
  • Automate and verify regular backups of PostgreSQL.

Volumes & ports

Compose uses named volumes for persistence:

volumes:
    static_volume:  # Nginx static files
    media_volume:   # Media files
    postgis_data:   # PostgreSQL data

External ports

  • Port 80 — Nginx reverse proxy (HTTP)

Monitoring

Tail service logs with:

sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml logs -f web
sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml logs -f nginx
sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml logs -f postgis

Maintenance

  • Stop services: 
    sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml down
  • Restart without rebuild: 
    sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml restart
  • Backup database: 
    sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml exec postgis pg_dump -U <your-db-user> <your-database-name> > backup.sql
  • Restore database: 
    sudo docker compose -f docker-compose.yml -f docker-compose.gcloud.yml exec postgis psql -U <your-db-user> <your-database-name> < backup.sql
