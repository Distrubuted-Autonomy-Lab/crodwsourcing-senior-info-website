Development environment setup

Quick-start setup for developers new to CareBot. Primary goal: get a working local instance using Docker.

Prerequisites

  • Docker Desktop (recommended for macOS/Windows)
  • Git
  • Browser (Chrome/Firefox/Safari)
  • Optional: Python 3.x locally (only if you run tools outside containers)

Clone the repositories

  • Application: carebot-deployable-crowdsourcing
  • Project website: website-crowdsourcing

The website is static HTML under website-crowdsourcing/docs/ (GitHub Pages).

First run (Docker Compose)

From the app repo’s root/ directory:

docker compose -f docker-compose.yml -f docker-compose.local.yml up --build

Then open http://localhost:8000/.

First-time admin user (staff login)

Create a staff user from the running containers (command names can vary by Compose setup):

docker compose -f docker-compose.yml -f docker-compose.local.yml exec web \
  python manage.py createsuperuser

If web is a different service name in your Compose file, use that instead.

Common setup issues (and fixes)

  • Compose run from the wrong directory: run from root/ to ensure Compose can locate override files and env.
  • Missing schema columns from older DB dumps: if you see errors like a missing resource_listings.title column, your local DB is out of sync with current Django models. Fix by applying the documented ALTER TABLE from Sprint 2 technical notes, or recreate local volumes.
  • AI chat not working: ensure GOOGLE_API_KEY is set (see Run & deploy and FAQs).
