Periodic cybersecurity review checklist

Use this list for ongoing reviews (quarterly or after major releases).

AreaReview item
SecretsNo API keys, DB passwords, or SECRET_KEY in Git; rotate after incidents or staff turnover.
TransportHTTPS enforced in production; HSTS where applicable; valid certs.
AuthStrong staff passwords; session timeout; review unused accounts.
OAuth (if added)Redirect URI allowlist; minimal scopes; secure storage of client secret; token refresh and logout tested.
App configDEBUG=False in prod; ALLOWED_HOSTS and CSRF_TRUSTED_ORIGINS match real hosts.
DatabaseNetwork isolation; least-privilege DB users; backups encrypted/restorable.
DependenciesReview requirements.txt / image bases for CVEs; patch cadence.
Public endpointsRate limiting on chat/submit; webhook secrets where used (e.g. Twilio).
Third-party AIGemini/OpenAI: data sent to vendor documented; quotas and logging; sponsor approval for sensitive topics.
ContainersNon-root users in prod images where feasible; minimal image attack surface.
LoggingNo PII in logs; access logs for moderation actions where required.
IncidentContact path and basic response steps (revoke keys, rotate secrets).
