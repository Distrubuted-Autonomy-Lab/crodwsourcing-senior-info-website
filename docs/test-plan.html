Test plan

This page documents manual test cases and a lightweight testing script for the core CareBot flows.

Scope

  • Public flows: browse, submit a resource recommendation (crowdsourcing)
  • Staff flows: login, view moderation queue, approve, merge, reject
  • Similarity: “find duplicates” behavior for moderation and submission review
  • Optional AI chat: graceful behavior with and without API key

How to run automated tests

From inside the running app container (or a local venv if supported in your setup):

python manage.py test

If your local DB uses unmanaged GIS tables, document limitations and rely on the manual cases below for end-to-end confidence.

Manual test cases (critical paths)

IDScenarioStepsExpected result
TC-001 Public submission creates a queued item Open the public submit page → fill required fields → submit Success message shown; item appears in moderation queue for staff
TC-002 Staff login required Try to open moderation routes while logged out Redirected to login; no sensitive moderation data is shown
TC-003 Approve submission Login → open moderation detail → approve Submission leaves queue; new listing is created/visible in catalog (or referenced as approved)
TC-004 Reject submission Login → open moderation detail → reject with reason (if supported) Submission marked rejected; it does not appear in public catalog
TC-005 Merge submission into existing listing Login → open moderation detail → choose an existing resource → merge Existing listing updated according to “smart merge” rules; submission closed
TC-006 Similarity search returns plausible duplicates On submission or moderation, run similarity for a known listing name / address Top results include the known listing(s); scores decrease with relevance
TC-007 Chat behavior without API key (if chat is exposed) Unset GOOGLE_API_KEY → open chat → send a message User sees a safe error / disabled state; server does not crash

Testing script (repeatable checklist)

  1. Bring up the stack: docker compose … up --build
  2. Verify homepage loads
  3. Run TC-001 through TC-006
  4. Run python manage.py test

If your team adds a shell script (example: scripts/retest-local.sh) in the app repo, link it here.
